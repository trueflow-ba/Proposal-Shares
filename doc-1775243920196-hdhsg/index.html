Platform Stack Overview

ServiceTitan CRM remains the system of record for all booking requests; nothing about that changes.
Clarity VoIP handles call routing, and the two architecture options differ in when and how the AI enters the call path. Under Option A (AI as Backup to Human), calls arrive via the ServiceTitan tracking number, pass through the Clarity main number to a live human receptionist during business hours, and reach the AI only on overflow; after hours, the Clarity main number routes directly to the AI. You retain full and independent authority to revoke that access at any time.

Zapier Middleware: Detailed Workflow Logic

The Zapier layer operates on a structured trigger-filter-action pattern and is managed entirely within TrueFlow's Zapier environment on your behalf. The ServiceTitan Zapier App Registration is created directly from within your ServiceTitan Developer Settings portal, meaning the OAuth credentials are generated and owned by your organization. Because you are the originating source of those credentials, you hold unconditional revocation authority independent of any action by TrueFlow. The Zap scope is restricted to booking request creation only; the integration has no read or write access to existing customer records, invoices, or any other ServiceTitan data. The detailed step-by-step logic is documented in full in Appendix Safeguard 3.

TRIGGERAI interaction completes (any path, any architecture option) → Full data package sent to Zapier webhook (name, phone, ZIP, service type, preferred windows, action taken: [Booking / CallBack / Transferred / Out-of-Area], timestamp)
FILTER 1Is caller ZIP in confirmed service territory? NO → Skip ServiceTitan record creation; real-time notification still fires (flag: "Out-of-Area Inquiry — No Booking Created")
FILTER 2Is action = Booking Request AND data complete? YES → BOOKING REQUEST (Action A, flag: "Standard Booking")
FILTER 3Is action = Schedule Call-Back? YES → BOOKING REQUEST (Action A, flag: "Call-Back Requested", preferred callback window included)
FILTER 4Was call transferred to live agent? YES → BOOKING REQUEST (Action A, flag: "Transferred — Context Attached", all collected data included)
ACTION ABOOKING REQUEST: Create ServiceTitan Booking Request, all fields populated (name, address, service type, preferred windows, action flag, source: AI Receptionist)
ACTION BRegardless of path → Fire real-time notification to Email + Google Chat with full booking summary (name, phone, ZIP, service, action taken, preferred windows)
LOGGINGAll Zap runs stored in Zapier run history for 30 days with full input/output visibility — auditable at any time

Appendix: Operational Safeguards & Architecture Detail

Three independent controls, because you should never feel locked into any system, at any time.

🔴 Safeguard 1 — The Phone Kill-Switch

The AI receptionist intercepts calls through call forwarding rules within your Clarity VoIP configuration and, depending on the chosen architecture, your ServiceTitan tracking number routing. Your primary Mr. Handyman business number never changes, is never ported, and is never dependent on TrueFlow infrastructure. The AI answers calls only because a routing rule has been configured to direct them there, and every one of those rules is a single toggle that you control independently.

Option A — AI as Backup to Human: The AI is active in two distinct windows. During business hours it activates only on overflow from the human receptionist; after hours it is the primary handler via the Clarity main number. To kill business-hours AI coverage, log into the Clarity VoIP admin panel and disable the overflow forwarding rule that routes to the AI endpoint. To kill after-hours AI coverage, update the after-hours forwarding rule in Clarity VoIP to point to voicemail or an after-hours message instead of the AI number. Each change takes fewer than 60 seconds and takes effect on the very next inbound call.

Option B — AI Takes All Calls First: The ServiceTitan tracking number routes directly to the AI receptionist number at all hours. To disable AI answering entirely, update the forwarding destination of that tracking number within ServiceTitan — or update the corresponding Clarity routing rule — to point directly to the Clarity main number or voicemail. This takes fewer than 60 seconds and takes effect immediately on the next inbound call.

  • Zero impact on ServiceTitan CRM — no records are altered, deleted, or locked
  • Zero impact on Zapier workflows — they simply stop receiving trigger data and go dormant
  • Callers experience no outage, no error message, and no disruption to service
  • All routing rules can be re-enabled instantly at any time without any reconfiguration
🔵 Safeguard 2 — The API Kill-Switch

The integration between TrueFlow's managed Zapier environment and ServiceTitan is authenticated via a Zapier App Registration that you create directly within your own ServiceTitan Developer Settings portal. This registration generates a dedicated Client ID and Client Secret using the OAuth 2.0 protocol. These credentials are the only authorized pathway for Zapier to write data into ServiceTitan. Credentials are maintained on file by TrueFlow as part of the active service engagement to support ongoing build and automation work. Because you are the originating source of these credentials within your own ServiceTitan instance, you hold unconditional revocation authority independent of any action by TrueFlow.

To disable all Zapier-to-ServiceTitan activity: Navigate to ServiceTitan → Settings → Developer → App Registrations → locate the TrueFlow Zapier registration → click Revoke or Rotate the Client Secret. This action takes effect in real time, immediately and permanently invalidating all Zapier authentication attempts. Zero ServiceTitan data is modified or deleted by this action.

  • Revocation is instantaneous — no propagation delay, no grace period
  • All existing ServiceTitan booking records and customer data are completely unaffected
  • API scope is limited strictly to booking request creation only — no access to existing records, invoices, dispatching, or any other ST module
  • A new Client Secret can be issued at any time to re-establish the integration with zero reconfiguration of conversation flows or AI logic
⚙️ Safeguard 3 — Zapier Middleware: Complete Architecture Breakdown

The Zapier layer is the connective tissue between the AI receptionist and ServiceTitan. Note that call-forwarding actions (forward to specific rep's number, forward to call queue) are executed in real time via Clarity VoIP during the live call; the Zapier layer captures the interaction data simultaneously and creates the appropriate ServiceTitan Booking Request regardless of which routing outcome occurred. All outcomes — standard bookings, call-back scheduling requests, and transferred calls — are recorded as Booking Requests differentiated by an action flag.
